## Overview

The authentication system ensures that only verified users can interact with the WhatsApp bot. It uses a One-Time Password (OTP) mechanism to verify user identity on their first interaction.

### Key Features

- **OTP Verification**: 6-digit codes sent via WhatsApp
- **Session Management**: 7-day authentication sessions
- **Automatic Expiry**: OTPs expire after 5 minutes
- **Rate Limiting**: Maximum 3 attempts per OTP
- **Twilio Signature Validation**: Ensures requests come from Twilio
- **Database Persistence**: All auth states stored in PostgreSQL

## Database Schema

### Existing Tables

#### `users` - User Profile Information
```sql
CREATE TABLE users (
    user_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    email VARCHAR(255),
    phone_number VARCHAR(20) UNIQUE NOT NULL,  -- Format: +1234567890
    full_name VARCHAR(255),
    hashed_password VARCHAR(255),
    password_history JSONB DEFAULT '[]'::jsonb,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

#### `userchats` - Conversation History
```sql
CREATE TABLE userchats (
    chat_id VARCHAR(50) PRIMARY KEY,
    customer_id UUID NOT NULL,
    is_active BOOLEAN DEFAULT true,
    messages JSONB DEFAULT '[]'::jsonb,  -- Array of {role, content, timestamp}
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES users(user_id) ON DELETE CASCADE
);
```

### New Authentication Tables

#### `user_auth` - Authentication Status
```sql
CREATE TABLE user_auth (
    auth_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID NOT NULL,
    phone_number VARCHAR(20) NOT NULL,
    is_authenticated BOOLEAN DEFAULT false,
    authenticated_at TIMESTAMP WITH TIME ZONE,
    expires_at TIMESTAMP WITH TIME ZONE,      -- Session expiry (7 days)
    last_activity_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE
);

CREATE INDEX idx_user_auth_phone ON user_auth(phone_number);
CREATE INDEX idx_user_auth_user_id ON user_auth(user_id);
CREATE INDEX idx_user_auth_expires_at ON user_auth(expires_at);
```

#### `otp_requests` - Pending OTP Verifications
```sql
CREATE TABLE otp_requests (
    otp_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    phone_number VARCHAR(20) NOT NULL,
    otp_code VARCHAR(6) NOT NULL,             -- 6-digit code
    is_verified BOOLEAN DEFAULT false,
    attempts INT DEFAULT 0,                   -- Failed attempts counter
    max_attempts INT DEFAULT 3,
    expires_at TIMESTAMP WITH TIME ZONE NOT NULL,  -- OTP expiry (5 minutes)
    verified_at TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_otp_phone ON otp_requests(phone_number);
CREATE INDEX idx_otp_expires_at ON otp_requests(expires_at);
CREATE INDEX idx_otp_is_verified ON otp_requests(is_verified);
```

## Security Considerations

### 1. **Twilio Signature Validation**
Every incoming request is validated using the `X-Twilio-Signature` header:
- Prevents spoofed requests
- Ensures requests come from Twilio
- Uses HMAC-SHA1 with your auth token
- Returns 403 Forbidden for invalid signatures

### 2. **OTP Security**
- **6-digit codes**: Balance between security and usability
- **5-minute expiry**: Short-lived to prevent replay attacks
- **Max 3 attempts**: Prevents brute-force attacks
- **One-time use**: OTPs marked as verified after successful use

### 3. **Session Management**
- **7-day sessions**: Balance between security and user experience
- **Activity tracking**: `last_activity_at` monitors user engagement
- **Automatic expiry**: Sessions expire after 7 days
- **Re-authentication required**: Users must verify again after expiry

### 4. **Database Security**
- **Prepared statements**: All queries use parameterized SQL
- **Foreign keys**: Cascading deletes maintain referential integrity
- **Indexes**: Fast lookups prevent timing attacks
- **Timestamps**: All operations logged with timezone

### 5. **Rate Limiting Recommendations**
Consider implementing:
- **Per-user rate limits**: Max 5 OTP requests per hour
- **Global rate limits**: Prevent system abuse
- **IP-based throttling**: Additional layer at reverse proxy

### 6. **PII Protection**
- **Phone numbers**: Store in E.164 format (+1234567890)
- **No plaintext passwords**: Use hashed_password column
- **Minimal data collection**: Only store necessary information
- **GDPR compliance**: Users can be deleted (CASCADE)

### 7. **Audit Trail**
All tables include:
- `created_at` - When record was created
- `updated_at` - Last modification time
- `authenticated_at` - When user last authenticated
- `verified_at` - When OTP was verified

## Configuration

### Environment Variables

Add to your `.env` file:

```env
# OTP Configuration
OTP_CODE_LENGTH=6                    # Length of OTP code
OTP_EXPIRY_MINUTES=5                 # OTP validity period
OTP_MAX_ATTEMPTS=3                   # Max failed attempts per OTP

# Session Configuration
AUTH_SESSION_DAYS=7                  # Authentication session duration
AUTH_REQUIRE_OTP=true                # Enable/disable OTP authentication
AUTH_ALLOWED_NUMBERS=                # Comma-separated list (optional whitelist)
```

### Database Migrations

Create the tables using the provided SQL:

```bash
# Connect to database
psql -h localhost -U postgres -d whatsapp_bot

# Run migrations
\i database/migrations/001_add_auth_tables.sql
```

## Monitoring & Maintenance

### Useful Queries

**Check authentication status:**
```sql
SELECT
    u.phone_number,
    u.full_name,
    ua.is_authenticated,
    ua.expires_at,
    ua.last_activity_at
FROM users u
LEFT JOIN user_auth ua ON u.user_id = ua.user_id
ORDER BY ua.last_activity_at DESC;
```

**View pending OTPs:**
```sql
SELECT
    phone_number,
    otp_code,
    attempts,
    expires_at,
    created_at
FROM otp_requests
WHERE is_verified = false
AND expires_at > NOW()
ORDER BY created_at DESC;
```

**Cleanup expired OTPs:**
```sql
DELETE FROM otp_requests
WHERE expires_at < NOW() - INTERVAL '1 day';
```

**Expire old sessions:**
```sql
UPDATE user_auth
SET is_authenticated = false
WHERE expires_at < NOW()
AND is_authenticated = true;
```

### Scheduled Maintenance

Consider setting up cron jobs:

```bash
# Clean up expired OTPs daily at 2 AM
0 2 * * * psql -c "DELETE FROM otp_requests WHERE expires_at < NOW() - INTERVAL '1 day';"

# Expire old sessions daily at 3 AM
0 3 * * * psql -c "UPDATE user_auth SET is_authenticated = false WHERE expires_at < NOW();"
```

## Troubleshooting

### User Not Receiving OTP

1. Check logs for OTP generation
2. Verify Twilio account balance
3. Check WhatsApp sandbox status
4. Verify phone number format

### OTP Always Invalid

1. Check system clock synchronization
2. Verify OTP expiry settings
3. Check database timezone settings
4. Review OTP generation logic

### Session Expires Too Quickly

1. Verify `AUTH_SESSION_DAYS` setting
2. Check database timestamp timezone
3. Review session expiry query
